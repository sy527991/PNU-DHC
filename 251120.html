1. 배경

현황

  • • 전통문화연구회 동양고전종합DB(중국의 동양고전): 고전 번역자료를 XML로 한국고전번역원에 납품
  • • 성균관대 한국경학자료시스템(조선시대 경학 주석 DB): 현재 한국고전번역원으로 이관
  • • 한국고전번역원 한국고전종합DB(한국문집총간 등): 경학자료의 출처가 대부분 한국 문집

필요성

  • • 한국고전번역원: 동양고전과 경학자료 두 자료의 연동 필요
  • • 동양고전 원문 + 조선시대 주석 통합 활용
  • • 국제 표준(TEI) 준수로 상호운용성 확보: 동아시아 경학자료로 확장 가능

2. 주요 기관

전통문화연구회

  • • 1991년 설립, 동양고전 전문 번역 기관
  • • 현재까지 341책 번역, 목표 1,500책
  • • 동양고전종합DB 운영: 모든 번역서를 XML 형식으로 웹 서비스
  • • URL: https://db.cyberseodang.or.kr

한국고전번역원

  • • 한국고전종합DB 구축 국가기관
  • • 특수고전협동번역사업 발주처
  • • 2024년 한국경학자료시스템 인수
  • • URL: https://db.itkc.or.kr

한국경학자료시스템 (성균관대 동아시아학술원)

  • • 『한국경학자료집성』디지털화 (145책)
  • • 사서오경에 대한 조선시대 주석
  • • URL: http://koco.skku.edu

3. 특수고전협동번역사업

사업 개요

한국고전번역원이 발주하고 전통문화연구회가 수행하는 동양고전 번역 사업

번역 대상

  • • 논어, 맹자, 시경, 서경
  • • 관자, 순자, 한비자
  • • 사기, 한서 등

결과물

  • • 한문 원문
  • • 국역문
  • • 주석서 포함
  • • XML 형식

4. 한국경학자료시스템

조선시대 유학자들이 사서오경에 단 주석을 체계적으로 정리한 DB

사서오경

사서: 대학, 중용, 논어, 맹자
오경: 시경, 서경, 역경, 예기, 춘추

주석 특징

  • • 조선 유학자 해석
  • • 주희 편장 구성으로 정리해서 주석별 검색 가능
  • • 대부분 문집에서 발췌

5. 두 자료의 관계

동양고전DB
논어 원문 + 현대 국역 + 역주
↔
경학자료
논어 원문 + 조선시대 주석

왜 연계 가능한가?

  • ✓ 동일한 원문 텍스트
  • ✓ 동일한 편장 구조 (주희 주석)
  • ✓ 상호보완적

6. 현행 시스템 문제점

핵심 문제

두 시스템 모두 XML이지만 서로 다른 DTD 사용

동양고전DB: juntong.dtd
경학자료: kbible.dtd

결과: 상호 연계 불가, 통합 검색 불가

7. 해결 방안: TEI 표준

동아시아 고전 TEI 적용 사례

불교 경전 디지털화에서 TEI 표준이 성공적으로 적용됨

CBETA 電子佛典集成 (대만)

  • • 1998년부터 TEI 활용 불교 경전 디지털화
  • • 원문, 異本, 역대 주석, 현대 번역 통합
  • • 국제 협력: 미국, 일본, 한국, 태국, 노르웨이 등
  • • 경전 계층구조(卷, 品, 章) 명확히 표현

SAT 大正新脩大藏經 (일본)

  • • 도쿄대학 TEI 기반 대장경 디지털화
  • • 체계적 불교 경전 정리, TEI동아시아 분과회 운영

시사점: 동양고전과 경학자료도 TEI 준수로 국제적 상호운용성과 협력 기반 확보 가능

8. TEI 통합 스키마

3층 구조

1층: 원문
한문 경전 - 양쪽 DB 공통
2층: 번역문
동양고전DB 현대 국역
3층: 주석
경학자료 조선 주석

9. 설계 원칙

  • • 공통 식별자: 주희 편장 구조 기준
  • • 명시적 연결: 원문-번역-주석 관계
  • • 출처 표시: DB 출처 명확히
  • • 확장 가능: 향후 중국이나 일본의 경학 주석자료 추가

10. TEI/XML 스키마: 기본 구조

편(篇)과 장(章) 계층 구조

<div type="pian" n="1" xml:id="lunyu_01">
  <head xml:lang="zh">學而篇</head>
  <head xml:lang="ko">학이편</head>
  
  <div type="zhang" n="1" xml:id="lunyu_01_01" 
       corresp="#kbible_01_01">
    <head xml:lang="ko">제1장</head>
    ...
  </div>
</div>

핵심 요소

  • • div: 텍스트 구획 (편/장)
  • • head: 제목
  • • @type: 구획 유형
  • • @n: 순서 번호

식별자 체계

  • • @xml:id: 고유 식별자
  • • lunyu_01_01: 논어 1편 1장
  • • @corresp: 경학자료와 대응

11. TEI/XML 스키마: 원문과 번역

원문-번역 병렬 구조

<!-- 원문 -->
<div type="original" xml:lang="zh" source="#juntong_db">
  <ab xml:id="orig_01_01_01" corresp="#trans_01_01_01">
    <seg xml:id="orig_01_01_01_s1">學而時習之</seg>
    <seg xml:id="orig_01_01_01_s2">不亦說乎</seg>
  </ab>
</div>

<!-- 번역문 -->
<div type="translation" xml:lang="ko" source="#juntong_db">
  <ab xml:id="trans_01_01_01" corresp="#orig_01_01_01">
    <seg corresp="#orig_01_01_01_s1">배우고 때때로 익히면</seg>
    <seg corresp="#orig_01_01_01_s2">또한 기쁘지 아니한가</seg>
  </ab>
</div>

특징: seg 단위로 정밀한 대응 관계 설정, AI 학습용 병렬 코퍼스 생성 가능

12. TEI/XML 스키마: 주석 구조 (1)

주석 유형 분류 및 연결

<div type="commentary">
  <!-- 중국 주석 (주희) -->
  <note type="ch-commentary" 
        target="#orig_01_01_01_s1" 
        source="#juntong_db"
        resp="#zhuxi" 
        xml:id="comm_zhuxi_01">
    <label>朱熹註</label>
    <p xml:lang="zh">說悅同。學之為言效也。</p>
  </note>
  
  <!-- 한국 주석 (박세당) -->
  <note type="kr-commentary" 
        target="#orig_01_01_01_s2" 
        source="#kbible_db"
        resp="#parksedang" 
        xml:id="comm_parksedang_01">
    <label>朴世堂註</label>
    <p xml:lang="zh">○學雖可訓為效但只言效則...</p>
  </note>
</div>

13. TEI/XML 스키마: 주석 구조 (2)

주석 속성 상세

유형 속성

  • • @type: 주석 유형
  • - ch-commentary: 중국 주석
  • - kr-commentary: 한국 주석
  • • @subtype: 세부 분류 가능

관계 속성

  • • @target: 주석 대상
  • • @source: 출처 DB
  • • @resp: 저자
  • • @xml:id: 주석 ID

효과: 동일 원문에 대한 중국/한국 주석 비교 분석, 시대별 해석 변천 추적 가능

14. TEI/XML 스키마: standOff

standOff: 관계의 명시적 표현

본문과 분리된 영역에서 요소 간 복잡한 관계를 명확히 정의

<standOff>
  <!-- 원문-번역 관계 -->
  <linkGrp type="alignment">
    <link target="#orig_01_01_01 #trans_01_01_01" 
          type="translation"/>
  </linkGrp>
  
  <!-- 주석 간 관계 -->
  <linkGrp type="commentary-relation">
    <link target="#comm_parksedang_01 #comm_zhuxi_01" 
          type="responds-to"/>
  </linkGrp>
</standOff>
type="translation": 번역 관계
type="responds-to": 응답/논쟁 관계

15. 스키마 통합 구조 시각화

전체 구조 흐름

1. 편/장 구조 → div[@type="pian/zhang"]
2. 원문 → div[@type="original"] + ab + seg
3. 번역 → div[@type="translation"] + ab + seg
4. 주석 → div[@type="commentary"] + note[@type]
5. 관계 → standOff + linkGrp + link

핵심: 모든 요소가 ID로 연결되어 원문-번역-주석의 관계망 형성
